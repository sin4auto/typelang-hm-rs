抽象構文木（AST）

目的:

  • 構文解析結果を評価/型推論で共用できる中立的な表現に落とし込む。

設計ノート:

  • 構文シュガーはここでは扱わず、parser/repl 側で正規化する。
  • 数値の基数（IntBase）は表示や一部の最適化のために保持する。
  • 型式（TypeExpr）は注釈・シグネチャ用で、型システムの Type とは分離。
